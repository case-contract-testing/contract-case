Alters the ContractCase log level below this matcher. Useful for debugging.

This has no effect on matching.

Note that this log level matcher will be saved into the contract, so it will
also affect the log level during verification. Usually you will want to
remove the use of this matcher before saving the contract.

<Tabs groupId="language">
  <TabItem value="typescript" label="Typescript">
```typescript
import { matchers } from '@contract-case/case-definition-dsl'

new matchers.convenience.ChangeLogLevel(logLevel: string, child: any)
```

#### Parameters <a name="Parameters" id="Parameters"></a>
| **Name** | **Type** | **Description** |
| --- | --- | --- |
| logLevel | `string` | The new LogLevel.<br/>One of "none" | "error" | "warn" | "debug" | "maintainerDebug" | "deepMaintainerDebug". see [LogLevel](https://case.contract-testing.io/docs/reference/configuring#loglevel-none--error--warn--debug--maintainerdebug) for details |
| child | `any` | The next matcher in the tree. |
